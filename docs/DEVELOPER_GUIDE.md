# راهنمای توسعه‌دهندگان سیستم مدیریت انبار

## مقدمه
این راهنما برای توسعه‌دهندگانی است که قصد دارند سیستم مدیریت انبار را توسعه داده یا تغییراتی در آن ایجاد کنند. در این مستند، ساختار پروژه، ارتباط بین ماژول‌ها، و نحوه توسعه آن شرح داده شده است.

## پیش‌نیازهای توسعه
- PHP 7.4 یا بالاتر
- MySQL 5.7 یا بالاتر
- IDE پیشنهادی: Visual Studio Code یا PhpStorm
- Git برای کنترل نسخه
- Composer برای مدیریت وابستگی‌ها

## ساختار پروژه
```
Inventory_Listing-WEB/
├── admin/                  # فایل‌های مربوط به پنل مدیریت
├── assets/                 # فایل‌های CSS، JavaScript و تصاویر
│   ├── css/
│   ├── js/
│   └── images/
├── core/                   # هسته اصلی سیستم
│   ├── includes/           # فایل‌های PHP هسته سیستم
│   └── api/                # API برای ارتباطات خارجی
├── includes/               # فایل‌های PHP مشترک
│   ├── functions.php       # توابع عمومی
│   └── theme_defaults/     # فایل‌های پیش‌فرض تم
├── themes/                 # پوسته‌های قابل تغییر سیستم
│   └── default/            # پوسته پیش‌فرض
├── docs/                   # مستندات پروژه
├── bootstrap.php           # فایل راه‌انداز سیستم
├── config.php              # تنظیمات سیستم
└── index.php               # نقطه ورود اصلی
```

## معماری برنامه
سیستم مدیریت انبار از معماری MVC-like استفاده می‌کند:
- **Model**: کلاس‌های مرتبط با دیتابیس و منطق تجاری در پوشه `core/includes`
- **View**: فایل‌های تم در پوشه `themes`
- **Controller**: فایل‌های PHP در پوشه اصلی که درخواست‌ها را پردازش می‌کنند

## راهنمای توسعه

### 1. تنظیم محیط توسعه
1. پروژه را از Git کلون کنید
2. فایل `config.example.php` را به `config.php` تغییر نام داده و تنظیمات را انجام دهید
3. دیتابیس را نصب کنید (از طریق `setup.php`)

### 2. ساختار تم
سیستم تم مبتنی بر فایل است. برای ایجاد یک تم جدید:
1. یک پوشه جدید در `themes/` ایجاد کنید
2. حداقل فایل‌های زیر را ایجاد کنید:
   - `functions.php` - تعریف توابع تم
   - `header.php` - قالب هدر
   - `footer.php` - قالب فوتر
   - `templates/` - پوشه برای قالب‌های صفحات مختلف

### 3. افزودن صفحه جدید
برای افزودن یک صفحه جدید به سیستم:
1. یک فایل PHP جدید در روت پروژه ایجاد کنید
2. از `bootstrap.php` استفاده کنید
3. کد زیر را به عنوان الگو استفاده کنید:

```php
<?php
// بارگذاری bootstrap
require_once __DIR__ . '/bootstrap.php';

// بررسی دسترسی
if (!$security->isLoggedIn()) {
    $_SESSION['redirect_after_login'] = basename(__FILE__);
    header('Location: login.php');
    exit;
}

// منطق صفحه
// ...

// نمایش قالب
get_template('صفحه_جدید');
?>
```

4. یک فایل قالب در `themes/default/templates/صفحه_جدید.php` ایجاد کنید

### 4. ماژول‌های سیستم

#### مدیریت امنیت
کلاس `SecurityManager` در `core/includes/SecurityManager.php` مسئول:
- احراز هویت کاربران
- کنترل دسترسی‌ها
- مدیریت جلسات
- ثبت رویدادهای امنیتی

#### مدیریت کالاها
فایل‌های مرتبط:
- `core/includes/Inventory.php` - کلاس اصلی مدیریت موجودی
- `core/includes/Product.php` - کلاس مدیریت کالاها

#### مدیریت گزارشات
فایل‌های مرتبط:
- `core/includes/Reports.php` - کلاس اصلی گزارش‌گیری

### 5. استایل کدنویسی
- از PSR-1 و PSR-2 برای استایل کد PHP پیروی کنید
- از توضیحات PhpDoc برای توابع و کلاس‌ها استفاده کنید
- متغیرها و توابع را به فارسی نام‌گذاری نکنید

### 6. استراتژی تست
- از PHPUnit برای تست‌های واحد استفاده کنید
- تست‌ها را در پوشه `tests/` قرار دهید
- برای هر کلاس یا ماژول اصلی، یک کلاس تست ایجاد کنید

### 7. مسیر توسعه آینده
- پیاده‌سازی API کامل
- بهبود رابط کاربری
- افزودن ماژول‌های:
  - مدیریت تأمین‌کنندگان
  - مدیریت سفارشات
  - هشدارهای موجودی
  - اتصال به سیستم‌های حسابداری

## حل مشکلات رایج توسعه

### 1. مشکل لود نشدن فایل‌های تم
بررسی کنید که:
- مسیر THEMES_PATH درست تنظیم شده باشد
- دسترسی‌های فایل صحیح باشد
- تابع theme_file به درستی کار کند

### 2. مشکلات دیتابیس
- از متدهای prepared statement استفاده کنید
- خطاهای دیتابیس را لاگ کنید
- تراکنش‌ها را برای عملیات‌های پیچیده استفاده کنید

### 3. مشکلات امنیتی
- از CSRF token در فرم‌ها استفاده کنید
- ورودی‌ها را پاکسازی کنید
- از prepared statements برای کوئری‌های دیتابیس استفاده کنید
